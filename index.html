ARDMS – Autonomous Racing Drone Motion Controller

Python • Autonomous • Firmware

i pursed creating ardms because of the lack of AM32 firmware in the united states due to the ukraine russia war that created the alka motors company. ardms is not a replacement for software like ardupilot or inav, it is simple, reliable and easy to use firmware that communicates with betaflight and its forks. ARDMS is an autonomous racing drone motion controller firmware built in Python to simplify drone control. It allows users to program autonomous flight routes with ease and repeatability. The system also includes automatic PID loop calibration for each motor: by strapping the quadcopter to a surface, ARDMS measures the power curves on each lateral axis to calculate optimal P, I, and D values for a pre-programmed PID controller.

view on github

Features

  • program custom flight routes with installed gps system
  • automatic pid calibration using strapped testing for initial setup
  • communicates with betaflight software and forks of it
  • supports esc communication using mosfet regulation
  • utilizes gps module to plan routes using a simple line based following

Process

  • compared ardupilot with inav to determine what my feature list would consist of
  • used shoelace theory to determine the area selected by the user
  • Implemented automatic PID calibration by calculating optimal P, I, and D values from the measured curves.
  • Built integration with Betaflight software for testing, debugging, and visualizing motor outputs safely without arming the ESCs.
  • Tested MOSFET activation and motor responses, iteratively refining the calibration algorithms for accuracy and stability.
  • Added functionality to allow users to program autonomous routes using a simple, repeatable workflow.

Details

  • chose mosfet regulation because am32 esc's require mosfet regulation to efficiently regulate the motor output. i had to add the mosfet regulation because regular f405 stack firmware distributed in the united states did not support mosfet regulation.
  • communication protocol of dshot was chosen because it gave the most speed between the stack and allowed for mosfet regulation as i wanted
  • i used full duplex isntead of a half duplex serial connection because i needed to support the mosfet regulation that took place over the dshot protocol which half-duplex did not have the bandwidth to communicate
  • installation via the betaflight firmware flasher using the firmware provided in the github

Accelerated 3D Printer

Cluster • Offload • 3D Printing

i decided to create a high speed 3d printer purely out of curiousty as i wanted to see how far i could push a ender 3 clone. so i decided to create a system that would work with the inbuilt printer hardware and give extra power to the printer that worked by the internal mcu controlled the motors and read limit switches, while computationally heavy tasks are offloaded to a minnowboard or raspberry pi cluster. this allows for faster printing without overloading the printer’s internal controller.

Features

  • 508% increase in printing speed
  • physical cooling improvement for high-speed extrusion
  • web interface to control cluster and send gcode
  • automatic mmu control for multi-material printing
  • theoretically infinite filament options during printing

Process

  • stripped the printer mcu and wired it to communicate with the computation device (minnowboard or 5-node raspberry pi cluster).
  • modified the printer mcu to offload computationally heavy tasks to the external cluster.
  • performed homing and identified communication latency issues, which were resolved by optimizing the protocol and increasing baud rate.
  • tested extrusion at higher speeds encountered cooling issues and i implemented a high-efficiency cooling system with intake and outtake fans over the build plate. which is shown in the second image

Details

  • i chose offloading computation over changing the printer board and parts becuase it was the most efficent and price effective solution to the slow speed.
  • added a high-efficiency cooling system with intake and outtake fans over the build plate because the extrusion wasn't being cooled on the build plate properly.
  • i chose to use a mac mini to emulate i(any number i wanted) number of pis to understand what was the optimal configuration for a effiecency and price.
